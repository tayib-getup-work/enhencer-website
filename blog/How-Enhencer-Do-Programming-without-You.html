How Enhencer Do Programming without You?
Read Article
Olcay Öztürk
05 December 2019
5 MIN READ



Nature of programming is human construction and human interpretation. However, in recent years, we are observing either a change in programming as a concept or a total deviation in the nature of it.

Before start reading, one must understand that computer programming is not a product, it is a concept. That is, in most of the cases, the main aim is not computer programming. Programming itself is a method to reach the main aim. Therefore, this concept can be utilized in almost every area of life in which digitalization is possible. In fact, digitalization process became so rapid in recent years that, almost everything has been affected by computer programming to some extent. Calculations in business is no exception. Programming before the era of artificial intelligence and machine learning was simple and to-the-point. Without the libraries and heavy computational power, programming was used as an accelerator for some tasks such as mathematical calculations and word processing. However, after 1950’s things got serious and first compiler languages were started to use. That is, instead of using direct mathematical symbols, these languages were using instructed symbols for that language so even though the human eye cannot understand it directly, a computer could.

During the early ages of programming, human intervention parameter was huge. Not surprisingly, programming languages were formed according to human needs, and human capabilities as well as technological capabilities. When talked about the automation of any kind, programming as a concept had not been automated but its products benefitted from automation. Meaning, programming was still a manual work, but output generation process, normally manual, was automated via programming itself. Thus, even though one had to encode the language to program, it was working its way up in the automation. So, how come we arrived at the discussion of automation of the concept? Well, if we wind the tape a little further, we would see artificial intelligence discourse in the realm of programming also. That means, early programmers were enthusiastic about automating both the product and the concept.

Latest artificial intelligence and machine learning paradigm proved that, programming as a concept can be automated too. Although not in a complete manner, a somewhat comprehensive automation of programming is possible. Last few years showed us that, the correct question in information technologies is “is it necessary?” not “is it possible”. Everything pioneers of programming had dreamed of became real to some extent. Hence, rather than looking for possibility, the best way to enhance the process is to put everything in hierarchical order to see priorities.

Automation of programming is necessary from every extent, this for sure. Main reason of it lies in the changing financial and demographic structure of the world. Now, everything is getting cheaper while average wage in the world decreases (contradicting with classic economic theory). This is because increasing mass-produced products and increase in population. Thus, while companies want more profits, people want to consume more. In order to reach an equilibrium, companies must shift their production technologies. That is, rather than going for an arithmetical increase they must go for a geometrical increase, which can only be attained via improvement in technology. Hence, in the age of digital transformation, a company, in order to reach equilibrium, must use its human capital for more communicational and creative tasks while repetitive mundane tasks must be performed with capable computers. This is where “doing computer programming without human” gets important.

Machine learning or other financially applicable programming methods are widely used in the financial sector to create value. However, manually creating and managing them via a team of experts is not the efficient way to go. Enhencer, frees you from that concern. Its AutoML feature works its way up to model creation and model selection; thus, without needing an army of experts, concept of programming can be automated to get the product, which is also a result of the automated process.
