Group By

Group By allows the user to see the direct effect of an independent variable to the target variable. In other words, users can see the direct effect between two variables with Group By function




A drop-down menu will appear after clicking Group By. The drop-down menu offers all the independent variables from the data that the user can select to observe the group by effects. Users can select any variable they want. After a variable is chosen, the output will look a lot like this;


  1. The variable name inside the group by box represents the independent variable selected for the Group By function.
  2. The Target Choice drop-down menu provides the choices form the target variable. If a target choice is selected, the respective target choice will be highlighted in the diagram shown in the middle panel.
  3. The Number in the right corner represents the sample size of the selected segment. Segment size shows how many individual records fall into that segment.
  4. The right panel shows the choices for the variable selected for the group by task.

Interpretation:

Why do we need Group By and what is it in the first place?

Let’s try to figure out; Does the age of the customer effect churn? In such a case our target variable is the Customer Status. Also remember that customer status has two distinct values, Churn & Retention, meaning a customer is either churned or not. Let’s choose Churn from the “Target Choice” dropdown.

As the group by variable let’s choose “Age” from the drop-down menu. You will see age is divided into 3 groups automatically by Enhencer. Enhencer does this to find the most optimal thresholds.

On the left, you will see a stacked bar chart that displays the group by results.

Notice the overall churn proportion in the data is 25%. It seems that customers aged between 52-68 are most likely to churn compared to other age groups.

This is how Group By helps the user to see how one particular variable can affect their target variable.
